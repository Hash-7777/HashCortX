Local host setup (Ubuntu 24.04). Ollama runs as a systemd service with override at /etc/systemd/system/ollama.service.d/override.conf setting OLLAMA_HOST=0.0.0.0 and OLLAMA_ORIGINS=* (the latter is required because HashCortx loads from file:// → Origin null). For images use a vision model, e.g. ollama pull llava.

Firewall (UFW). sudo ufw allow 22/tcp · sudo ufw allow 11434/tcp · sudo ufw enable. Verify with sudo ufw status verbose.

Wi-Fi & LAN. Both via NetworkManager. Wi-Fi: nmcli radio wifi on · nmcli dev wifi list · nmcli dev wifi connect "<SSID>" password "<pwd>". To reserve a stable IP, set a DHCP reservation on the router for your host's MAC, or pin it: nmcli con mod <name> ipv4.addresses 192.168.1.107/24 ipv4.gateway 192.168.1.1 ipv4.dns "1.1.1.1 8.8.8.8" ipv4.method manual.

Web search options (any one of these works — Tavily is simplest):
· Tavily (recommended) — free 1000 searches/mo, sign up at tavily.com, paste the tvly-… key above.
· Google CSE — free 100/day, key from console.cloud.google.com + Search Engine ID from programmablesearchengine.google.com.
· No keys → research agents fall back to Wikipedia.

NVIDIA NIM (optional cloud brain). Free tier at build.nvidia.com — sign in, generate an nvapi-… key, paste it above. Used by the auto-router when your question needs a bigger general-knowledge model than the local coder. Only the messages on the routed turn are sent — never your full chat history.

Cloud Models (direct, free). Add API keys for Groq (console.groq.com), Google AI Studio (aistudio.google.com), or OpenRouter (openrouter.ai). They appear in the model dropdown under labelled groups. Select one to talk to that service directly — the local host doesn't need to be online. A ☁ CLOUD badge appears in the topbar so you always know when messages leave your device.

Auto-router. When enabled, picks the backend per message based on topic (code → local coder, news/recency → NVIDIA + Tavily search, science/medical → PubMed + local). A chip above the composer shows which backend will handle the next send — click it to override for one message. Toggle Privacy: Local only to block auto-cloud routing; you'll also be asked to confirm any time you manually pick a cloud model.

Privacy data flow.
· Chat history, settings, API keys → stored in browser localStorage only. Never sent anywhere unless you pick a cloud model or agent tool.
· Local model → messages go to your own machine at 192.168.1.107 only. Zero external traffic.
· Groq / Gemini / OpenRouter / NVIDIA → only the current turn context (up to your Memory Depth setting) is sent to that API, over HTTPS, with Referrer-Policy: no-referrer. No browsing history, no other chats, no system info.
· Web search tools (Tavily, Google CSE) → fire only when the Inject button is enabled and the router is on a web-search route. Query text only, no chat history.
· Timezone → resolved entirely in-browser (Intl.DateTimeFormat), no network call.
· No analytics, no telemetry, no ads, no tracking of any kind.

Query rewriter. Pick a small fast model (e.g. llama3.2:3b or qwen2.5:3b) — when an agent with tools is active, this model first rewrites your conversational message into a short keyword search query, then the browser fires the API call. Makes the Researcher and Papers agents much sharper on casual questions. Leave off to search with your raw message.
